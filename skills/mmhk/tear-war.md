# 终局篇——亚沙之泪与符文要塞

## 概述

世界运行一段时间后（默认60天）进入终局阶段。终局有多种类型，核心是联盟之间争夺控制权，最终决定世界胜利者。

## 终局类型

| 类型 | 说明 | 是否终结世界 |
|------|------|-------------|
| TEAR_WAR | 亚沙之泪战争——联盟收集足够泪之后触发 | 是 |
| RUNIC_FORTRESS_WAR | 符文要塞战争——控制最多符文要塞的联盟获胜 | 否 |
| DWARVEN_ELECTIONS | 矮人选举——基于符文要塞控制权的竞争 | 否 |
| DARKELVES_INVASION | 暗精灵入侵——替代终局场景 | 否 |

## 亚沙之泪战争 (Tear War)

### 触发条件

1. 世界运行时间超过 `startWorldDuration`（默认 5,184,000 秒 = 60 天）
2. 某个联盟收集的泪数量达到阈值（默认 13 颗）
3. 泪通过占领含有"圣杯建筑"(Grail Structure) 的区域获得

### 流程

```
联盟泪数 ≥ 13 → 创建终局动作(TYPE_END_GAME) → 倒计时结束 → 世界终结
```

- 触发时向全服广播警告消息
- 如果联盟泪数跌回阈值以下，终局动作取消
- 终局完成后世界进入 `ended` 状态，显示排行榜和奖励

### 泪的获取

- 泪 (Tear of Asha) 从区域中的圣杯建筑获取
- 联盟累计泪数记录在 `Alliance.cumulTear`
- 联盟排行榜使用泪数排名

## 符文要塞 (Runic Fortress)

符文要塞是地图上的关键区域建筑，控制权是终局竞争的核心。

### 结构

- 符文要塞有多层楼 (floor)
- 每层可以设置不同的防御类型（兵种配置）
- 攻击方需要逐层攻破

### 防御管理

**设置某层防御类型**：
```json
{"type": "runicFortressAction", "params": {
  "action": "CHOOSE_DEFENSE",
  "regionBuildingId": 5,
  "floorNumber": 1,
  "regionBuildingComponentEntityId": 12
}}
```

**重置攻防兵种**（有冷却时间）：
```json
{"type": "runicFortressAction", "params": {
  "action": "REROLL_ATTACK_DEFENSE",
  "regionBuildingId": 5
}}
```

### 控制权

- 联盟控制符文要塞后获得影响力和资源收益
- 其他联盟可以通过 `HERO_ATTACK_REGIONBUILDING` 攻击要塞
- 控制最多要塞的联盟在符文要塞战争中获胜

### 操作权限

- 只有联盟成员中有编辑权的玩家可以管理要塞防御
- 如果要塞门口有敌方英雄，无法执行防御操作
- 重置攻防有冷却时间限制

## 终局排行与奖励

- 终局触发时记录各联盟的泪数、符文要塞数、综合排名
- 获胜联盟成员获得特殊成就奖励
- 终局结果影响玩家的"血统"(Heredity) 系统——下一世界的初始加成

## 终局策略

1. **前期积累**：60天内尽可能发展实力、扩张领土、控制关键建筑
2. **联盟协作**：终局依赖联盟整体实力，单人无法赢得终局
3. **控制要塞**：优先占领和防守符文要塞，配置合理的防御阵容
4. **收集泪**：TEAR_WAR 类型下，优先争夺含圣杯建筑的区域
5. **防御配置**：根据对手的兵种倾向调整要塞每层的防御类型，利用兵种克制
