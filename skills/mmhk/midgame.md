# 中期发展篇

## 何时进入中期

**判据：`player.maxCityCount >= 2`。** 当任意一项排名等级（统治力/财富/荣耀）允许你拥有第 2 座城市时，进入中期。

中期的核心目标从"发展单城"转变为"扩张+多城协作"。你需要同时管理多座城市的建设、资源调度、英雄分配，并开始与其他玩家互动。

**重要：每座新城市的内部发展仍然参照 `$SKILL_DIR/development.md`（建筑优先级、开局流程等）。** 本文件聚焦于中期特有的跨城决策。

---

## 一、开城扩张（中期最重要的事）

### 1.1 何时开新城

检查 gameState：
```
if player.currentCityCount < player.maxCityCount:
    应该立即规划开新城
```

开城是中期最高优先级的动作。每多一座城市 = 多一套矿产收入 + 多一套建筑队列 + 多一个英雄槽位。拖延开城 = 落后于对手。

**开城前置条件：**
1. `currentCityCount < maxCityCount`（排名等级允许）
2. 有一个英雄携带部队（用于 HERO_SETTLE_REGION）
3. 有一座资源充足的城市发起（约 90k 金 + 90 木 + 90 矿会从该城市扣除）
4. 已用 `scanRegionMines` 找到合适的目标区域

### 1.2 选址策略（金矿是命脉）

**核心原则：没有金矿的区域不能发展，不要开城。**

金矿是城市经济的命脉，每个金矿提供 2000 金/天/级的收入。无金矿城市只能靠市政大厅的 2000~13000 金/天，根本无法支撑建筑、招募和维护费。

**选址优先级（严格按此顺序）：**

| 优先级 | 矿产配置 | 说明 |
|--------|---------|------|
| **第一** | 2 金矿 + 1 木矿 + 1 矿石矿 | 最优配置，完全自给自足 |
| **第一（替代）** | 2 金矿 + 2 木矿 或 2 金矿 + 2 矿石矿 | 偏科但仍是双金双普通 |
| **第二** | 2 金矿 + 阵营稀有矿 + 任意矿 | 金矿足够，稀有矿中后期有价值 |
| **第三** | 1 金矿 + 木矿 + 矿石矿 + 阵营稀有矿 | 勉强可用，发展较慢 |
| **不选** | 0 金矿 | **绝不开城**（End Game 抢神器除外） |

阵营主要稀有资源：圣堂=水晶、地狱=硫磺、学院=宝石、亡灵=水银。

**选址操作流程：**
1. `scanAreaMines`（w=15, h=15）— **一次性获取 15×15 范围内所有 Region 的矿产数据**（对应前端"采矿勘探"按钮）
2. 从返回的 `cells[]` 里筛选 `captured==false` 且 `type=="plain"`（**小写**）的平原
3. 按上表优先级打分（双金矿 > 金矿+稀有 > 单金矿 > 零金矿绝不开）
4. 如果 15×15 范围内无理想位置，用不同 `x`/`y` 多次调用扩展搜索
5. 从资源最充足的城市发起 `HERO_SETTLE_REGION`

**示例：**
```json
{"type":"scanAreaMines","params":{"w":15,"h":15}}
```
返回 `cells[]`，每个 cell 含 `regionId/x/y/captured/type/mines`。直接用这份数据打分，不需要再调用 scanRegionMines。

### 1.3 新城初始化三件套

开城后**立即建造**（参考 $SKILL_DIR/development.md 的开局流程，但重点是这三样）：
1. **TAVERN**（酒馆）— 解锁招募新英雄
2. **RECRUIT_T1**（招募建筑）— 开始补充兵力
3. **MARKETPLACE**（市场）— 开启商队运输

然后从老城商队运送 30k 金 + 10 木 + 10 矿，让新城有启动资金。

### 1.4 新城驻军控制

**绝对不要把大量部队搬到新城。** 驻军维护费会吞噬新城微薄的收入。

规则：
- 新城驻军总战力控制在 **30k-60k**（够打 NPC 就行）
- 配置：SHO×60 + INF×30 + CAV×20 左右（灵活配比）
- 打完 NPC 获得的中立兵种（弱单位）及时 DISBAND 或转移，不要白吃维护

## 二、多城资源管理

### 2.1 木材是中期最大瓶颈

大部分城市没有木矿（每座城只有 4 个矿槽，随机分配）。木材需求巨大（建筑主要消耗木+矿），但产量极低。

**应对策略：**
1. **优先开有木矿的城市**（选址时考虑）
2. **NPC 市场购买木材**（2000 金/个，贵但立即可用）
3. **商队运输**（从有木矿的城市运到缺木的城市）
4. **拍卖行求购**（CREATE_OFFER_REQUEST，价格比 NPC 更优）

### 2.2 商队运输策略

```json
{"type":"marketPlaceAction","params":{
  "action":"CREATE_TRADEROUTE",
  "regionId":"<发出城市regionId>",
  "destRegionNumber":"<目标城市regionId>",
  "resources":{"GOLD":0,"WOOD":15,"ORE":15,"MERCURY":0,"CRYSTAL":0,"SULFUR":0,"GEM":0},
  "repeatNb":10,
  "stockThreshold":1,
  "waitingTime":0
}}
```

**原则：**
- 从有矿的城市运到没矿的城市（不要反向运输）
- 普通资源（木/矿石）优先运输，金币各城自己产
- `repeatNb` 设最大值，`stockThreshold=1`（库存低时暂停，避免掏空源城市）

### 2.3 卖多余稀有资源换金

前中期稀有资源用处不大（T1-T5 招募只需金币），但占储存上限。

```json
{"type":"marketPlaceAction","params":{
  "action":"SELL_NPC","regionId":"<regionId>",
  "order":{"MERCURY":20,"SULFUR":20}
}}
```

把用不到的稀有资源卖给 NPC 换金，优先卖非阵营稀有（圣堂卖水银/硫磺，保留水晶）。

## 三、英雄管理（中期）

### 3.1 英雄分配原则

每座城市需要至少一个英雄。英雄空闲 = 浪费。

| 英雄角色 | 分配位置 | 职责 |
|---------|---------|------|
| 主力战斗英雄（ATK 高） | 新城 / 前线 | 清 NPC、PvP 攻击 |
| 副战斗英雄 | 翻转城市间 | 补充清 NPC、解放建筑 |
| 经济英雄（属性低） | 老城 | 升矿（IMPROVE_MINE）、建造野外建筑 |

### 3.2 英雄永远不空闲

空闲英雄检查清单（按优先级）：
1. 有未清的 NPC zone？→ 装兵去打
2. 有可升的矿（improveLevel < upgradeLevel-1）？→ 卸兵去升矿
3. 有未解放的建筑？→ 去解放
4. 新城需要英雄？→ 移动过去
5. 以上都没有？→ 建造野外建筑、学习职业/技能

**⚠️ IMPROVE_MINE 关键前提：英雄必须无部队且在同 region**
- 错误 `"hero should go without any army"` → 英雄身上有兵
- 错误 `"hero isn't in #XXX region (targeted)"` → 英雄不在目标矿所在城市
- 在城内 zone 间移动（格式 `"cityId;zoneId"`）时，**兵会留在英雄身上**
- 解决方案：先用 `unitStackListAction MOVE_FROM_REGION` 把兵卸到驻军，再升矿
- **满级矿打不动**：upgradeLevel=10 + improveLevel=9 已达上限，IMPROVE_MINE 会返回 `"this mine cannot be improved"`

**金矿升级优先级（实战经验）：**
- 主城/渊金城：金矿已满级（Lv10+9），无需再升
- 新城（金辉湾、金源城）：金矿 Lv1~3，是重点打磨对象
- 多个英雄可并行：各英雄分别移动到不同城市的矿区，同时升矿

### 3.3 酒馆招募新英雄

中期需要 4-6 个英雄（每城 1 个 + 机动英雄）。

- 定期对所有城市调用 `getContent TavernFrame` 检查酒馆
- 酒馆空时用 `tavernAction REROLL`（有冷却和金币消耗）
- 招募优先选 ATK 高 + heroTrainingEntityTagName 成长好的英雄
- 参考 `$SKILL_DIR/heroes.md` 查看各 training type 的满级属性分配

## 四、排行榜追赶

### 4.1 判断当前短板

每次登录读取 gameState 后：
```python
gap_dom = nextDominationObjective - dominationScore  # 统治力差距
gap_wealth = nextWealthObjective - wealthScore        # 财富差距
gap_honor = nextHonorObjective - honorScore           # 荣耀差距
```

差距最小的维度 = 最容易突破的，优先冲这个。突破排名等级 → 可能解锁更多城市 → 加速所有维度增长。

### 4.2 快速提升各维度

| 短板 | 最快提升手段 |
|------|------------|
| 统治力 | 开新城（新城建筑 + 部队全算分）、建筑队列永远满 |
| 财富 | 升矿（IMPROVE_MINE 是长期 ROI 最高的）、占更多矿区、开有金矿的新城 |
| 荣耀 | 高荣耀战斗（带少量兵打强 NPC，荣耀 > 1.0 = XP 加成）、多英雄并行战斗 |

## 五、联盟与外交

### 5.1 何时创建/加入联盟

**拥有 2 座城市后就应该建立联盟。** 联盟越早建立，影响力积累越快。

```json
{"type":"allianceAction","params":{
  "action":"CREATE_ALLIANCE",
  "allianceName":"联盟名称",
  "description":"联盟描述"
}}
```

### 5.2 邀请策略

- 用 `getContent WorldMap` 找到附近的其他玩家城市
- 用聊天（`chat` 动作）联系对方
- 对地理位置相近的活跃玩家发起邀请
- 同联盟成员的影响力叠加，有助于控制区域和终局战争

### 5.3 外交时机

- **早期（2-3 城）：** 专注发展，建联盟但不主动开战
- **中期（3-5 城）：** 通过联盟建立防御网络，侦察潜在威胁
- **中后期（5+ 城）：** 根据实力决定进攻或结盟，准备终局

## 六、PvP 时机判断

### 6.1 何时可以开始 PvP

PvP 不是越早越好。过早 PvP 会消耗大量部队（统治力下降）和资源。

**开始 PvP 的条件：**
1. 至少 3 座城市稳定发展
2. 有足够的兵力储备（主城驻军 > 100k 战力）
3. 已侦察到对手实力（用 HERO_SCOUT_CITY / HERO_SCOUT_TROOPS）
4. 对手明显弱于自己，或者对手威胁到自己的扩张路线

### 6.2 PvP 类型选择

| 操作 | 风险 | 收益 | 适用场景 |
|------|------|------|---------|
| 掠夺（HERO_PILLAGE_ZONE） | 低 | 偷资源 | 对方矿区资源丰富 |
| 侦察（HERO_SCOUT_*） | 极低 | 信息 | 任何时候先侦察 |
| 进攻（HERO_ATTACK_REGION） | 高 | 夺城 | 实力碾压对手 |
| 围城（HERO_SIEGE_*） | 很高 | 夺城 | 需要长期围困 |

**中期推荐：先侦察，再掠夺，不急着攻城。** 详细 PvP 操作参见 `$SKILL_DIR/pvp.md`。

## 七、中期检查清单

每次登录后按此顺序检查：

```
1. 读 gameState
2. 检查 maxCityCount > currentCityCount？→ 立即规划开新城
3. 建筑队列是否有空闲？→ 排满建筑（资源不够就买木材）
4. 英雄是否空闲？→ 分配任务（打 NPC > 升矿 > 解放建筑 > 移动到新城）
5. 检查三大排行短板 → 集中提升最弱维度
6. 检查联盟状态 → 邀请附近玩家、检查外交
7. 新城需要资源？→ 发商队或买 NPC 资源
8. 有多余稀有资源？→ 卖给 NPC 换金
```
