# 发展篇

## 战略总纲：三大排行指标

游戏的终极目标是三个排行榜分数。**所有行为都应服务于提升这三项分数。** 每次 gameState 检查 `player.dominationScore`、`player.wealthScore`、`player.honorScore`，空闲时优先提升最弱的维度。

### 统治力（dominationScore）— 你的帝国规模

**计算方式：** 城市建筑价值 + 野外建筑价值 + 军队总战力（实时计算，会涨会跌）

**提升手段（按效率排序）：**
1. **建造/升级城市建筑** — 每栋建筑贡献固定统治力，高级建筑贡献更多
2. **招募部队** — 军队战力直接计入统治力，招得越多分越高
3. **建造野外建筑** — 在已占领的 Zone 上建造，提供额外统治力
4. **开新城** — 新城市的建筑和部队都计入，是统治力倍增器
5. **占领更多 Zone** — 占领本身不加分，但为野外建筑提供位置

**注意：** 部队战损会降低统治力。打败仗不仅浪费兵力，还直接扣分。

### 财富（wealthScore）— 你的累计经济产出

**计算方式：** 历史累计资源收入总和（只增不减，花掉资源不扣分）

**提升手段（按效率排序）：**
1. **升级矿产（IMPROVE_MINE）** — 提高产出 = 每天更多财富增长，复利效应
2. **占领有矿的 Zone** — 越多矿产 = 越多收入来源
3. **保持城市持续运转** — 资源收入是按时间累加的，一秒都不要浪费
4. **开新城** — 新城市的矿产收入独立叠加
5. **市场交易** — NPC 买卖本身不增加财富，但商队运输的资源收入计入

**关键认知：** 财富是时间函数。越早提升资源产出，长期差距越大。升矿的 ROI 是所有操作中最高的。

### 荣耀（honorScore）— 你的英雄战斗经验

**计算方式：** 所有英雄累计获得的经验值总和（只增不减）

**提升手段（按效率排序）：**
1. **高荣耀战斗** — 带少量兵利用克制优势打强敌，荣耀修正 > 1.0 = XP 加成（详见 $SKILL_DIR/combat.md）
2. **持续战斗不停歇** — 英雄空闲 = 浪费经验，永远安排下一场
3. **打更强的 NPC** — Lv3+ 的 NPC 给的经验远超 Lv1，即使需要更多准备
4. **多英雄并行** — 每个英雄独立获取经验，都累加到荣耀总分
5. **冥想/侦察** — 没有合适战斗目标时的替代经验来源

**关键认知：** 打输了一分经验都没有。宁可降低荣耀修正（多带兵）确保赢，也不要冒险打输。败方 XP = 0 + 损兵 = 统治力也跌。

### 三维度平衡决策

每次心跳循环检查三项分数，判断当前短板：

| 短板维度 | 优先行动 |
|----------|---------|
| 统治力低 | 建筑队列不能空、招募部队、建野外建筑 |
| 财富低 | 升矿（IMPROVE_MINE）、占领矿区 Zone |
| 荣耀低 | 安排英雄战斗，优化荣耀修正 |
| 三项均衡 | 按游戏循环正常推进，兼顾所有方面 |

## 开局流程（新玩家或新城市）

注册完成后，你的首要目标是尽快建立经济和军事基础。严格按以下顺序执行：

### 第一步：立即建造酒馆

注册后城市里有一栋预先解放的建筑——酒馆（TAVERN）。不需要英雄去解放，可以直接建造。

```
addAction: BUILD_CITYBUILDING, actorType=Region, actionParams="<酒馆建筑ID>;<速度>"
```

酒馆是最高优先级建筑，因为它解锁第二个英雄。没有第二个英雄，你只能用初始英雄一个一个解放建筑，效率极低。

### 第二步：酒馆建好后立即招募第二个英雄

酒馆建成后，通过 `getContent` 获取 `TavernFrame` 查看可招募的英雄列表。每个酒馆英雄自带约5000战力的军队——这是巨大的战力提升（初始英雄可能只有1000-2000战力）。

```
getContent: TavernFrame → 获取酒馆英雄ID
addAction: RECRUIT_HERO, actorType=Region, actionParams="<酒馆英雄ID>"
```

招募英雄后，你就有了两个可以并行执行动作的英雄。

### 第三步：用强力英雄解放城内建筑

城市中的建筑需要先解放才能建造。解放战的NPC大约有1500-2000战力，刚招募的酒馆英雄（约5000战力）可以轻松取胜。

```
addAction: HERO_LIBERATE_CITY_BUILDING, actorType=Hero, actionParams="<建筑ID>;<区域ID>"
```

解放优先级参考下面的「建筑优先级」章节。

### 第四步：每次解放后领取邮件奖励

每次成功解放城市建筑后，系统会发送一封 LIBERATE_CITY_REWARD 邮件，内含免费部队奖励。必须及时领取。

```
getContent: MessageBoxFrame → 找到未读奖励邮件
messageAction: action=collectReward, messageId=<邮件ID>, choiceId=<选项ID>
```

这些免费部队是前期兵力的重要来源，不要遗漏。

### 第五步：连续解放之间补充英雄兵力

每次战斗后英雄会损失部队。在发起下一次解放战之前，必须从城市驻军中补充兵力给英雄：

```
unitStackListAction: action=garrisonToHero, heroId=<英雄ID>
```

如果驻军中兵力不足，先招募新部队再补充。不要让英雄带着残兵去打解放战。

### 第六步：建筑队列不能空闲

城市在任何时刻都应该有建筑正在建造。如果当前建筑完成了，立即开始下一个。建筑是城市发展的核心，每一秒空闲都是浪费。

通过 `gameState` 检查 `cities[].buildingQueue`（数组，最多4个）：
- 空数组 = 队列空闲，应立即安排建筑
- 长度 < 4 = 还有空位，可以继续排队更多建筑
- 长度 = 4 = 队列已满，等待完成

建筑排队功能允许一次性排队最多 4 个建筑，无需等待前一个完成再手动安排下一个。

## 建筑优先级

按照以下优先级顺序建造和解放建筑：

### 1. 酒馆 TAVERN（最高优先级）
- 预先解放，直接建造
- 解锁第二个英雄，大幅提升并行能力

### 2. 市政大厅升级链 TOWN_HALL → CITY_HALL → CAPITOL
- 金币是最重要的资源，直接影响招募速度和建造速度
- 每级收入：VILLAGE_HALL(初始) → TOWN_HALL(4000/天) → CITY_HALL(7000/天) → CAPITOL(13000/天)
- 尽早升级到 CAPITOL 是长期经济的基础

### 3. 招募建筑 RECRUIT_T1 → T1P → T2 → ... → T7
- 每个兵种等级都需要对应的招募建筑先解放再建造，才能招募该等级的部队
- 解放建筑 → 建造招募所 → 招募部队，三步缺一不可
- 低级兵种（T1-T3）是前期的主力，优先解放和建造
- 升级版（T1P, T2P...）在基础版之后建造，解锁更强的进阶兵种

### 4. 其他建筑
- **市场 MARKETPLACE** — 提高资源库存容量（主要），附带买卖资源功能
- **铁匠铺 BLACKSMITH / ADVANCED_BLACKSMITH** — 生产 T8 弩车（攻城用），优先级最低，只有准备围城时才需要
- **魔法行会 MAGIC_GUILD_LVL1~5** — 英雄法术，后期重要
- **城防 FORT → CITADEL → CASTLE** — 给驻防部队提供大量防御加成，中后期重要
- **资源仓 RESSOURCE_SILO_LVL1~10** — 增加资源储存上限

## 英雄管理

### 核心原则：英雄不能空闲

英雄是游戏中最宝贵的资源。**空闲的英雄 = 浪费的时间。** 每个英雄在任何时刻都应该在做事：

- 解放建筑
- 攻击 NPC 区域
- 升级矿产
- 学习职业/技能
- 建造野外建筑
- 移动到新区域
- 巡逻防守

每次登录时检查所有英雄的 `_current_slaveActionId`，为 null 的英雄立即安排任务。建筑队列同理——`buildingQueue` 长度 < 4 时应继续排队建筑。

### 尽快招募更多英雄

英雄是你执行所有动作（解放、攻击、移动）的唯一载体。更多英雄 = 更多并行动作 = 更快发展。

每次酒馆刷新都检查是否有值得招募的英雄。通过 `getContent` 获取 `TavernFrame` 查看当前可招募列表。

### 酒馆刷新（REROLL）

如果当前酒馆英雄不理想（属性成长差、职业不合需求），可以花金币刷新：

```json
{"type":"tavernAction","params":{"action":"REROLL","regionId":"<区域ID>"}}
```

- 费用递增：第1次200金，第2次400金，第3次800金...（每次翻倍）
- 有冷却时间，不能连续刷新
- 刷新后用 `getContent TavernFrame` 查看新英雄
- 前期金币紧张时不要频繁刷新，除非当前英雄非常差

### 英雄解散

多余的英雄或者属性太差的英雄可以解散，腾出英雄位：

```json
{"type":"heroAction","params":{"action":"DISBAND_HERO","heroId":"<英雄ID>"}}
```

解散条件：
- 英雄身上不能有部队（先卸载到驻军）
- 英雄不能在执行动作
- 英雄不能在围城中
- 不能解散最后一个英雄

### 英雄改名

```json
{"type":"heroAction","params":{"action":"RENAME_HERO","heroId":"<英雄ID>","heroName":"新名字"}}
```

传奇英雄不可改名。

### 英雄职业选择

英雄最多学3个职业，选错无法撤销。**选职业前必须查 `heroTrainingEntityTagName`，对照 `$SKILL_DIR/heroes.md` 的满级属性表确认 ATK/DEF/MAG 分配。**

关键规则：
- **不要一口气学3个职业** — 技能点有限，先学1个职业把核心技能升到Lv2再考虑第2个
- **满级 MAG ≥ 40 的英雄必须学魔法职业**（魔法成长是稀缺天赋）
- 高 ATK + 高 MAG 的英雄是攻魔双修型：先学斗士升核心技能，再学魔法职业
- 经济职业（建造者/拓荒者/商人）只给满级 ATK 和 MAG 都 < 40 的非战斗英雄
- **前期多英雄职业多样化（重要）**：多个英雄不要全学同一种战斗职业（如全斗士）。应至少让一个英雄的第二职业选择**建筑师 ARCHITECT**（工长技能 +27% 建造/升矿速度），大幅减少等待时间。战斗英雄第一职业学斗士，第二职业学建筑师是前期发展效率最高的组合

学习职业的前提条件：
- 英雄必须有空闲技能点（`xpPoints > 0`）
- 英雄身上不能有部队（必须先移走）
- 英雄不能正在执行其他动作
- 最多学习3个职业

### 技能点系统

- 英雄初始有1个技能点
- 每次升级获得技能点
- 学习职业消耗1点
- 学习技能消耗按等级递增
- 技能点是稀缺资源，谨慎分配

### 学习流程

1. 通过 `unitStackListAction` 将英雄部队移回驻军
2. 通过 `getContent` 获取 `HeroFrame` 查看可用职业和技能
3. 通过 `addAction: HERO_LEARN_CLASS` 或 `HERO_LEARN_SKILL` 学习
4. 学习完成后将部队补充回英雄

不要查询具体技能数值——通过 `getContent` 获取 `HeroFrame` 可以看到所有可学习的职业和技能详情，包括效果描述和消耗。

## 每次登录的游戏循环

每次获取 gameState 后，按以下顺序处理：

### Step 1: 邮件与任务
- 检查 `unreadMessages` 是否 > 0，如有则获取 `MessageBoxFrame` 领取所有奖励
- 检查 `quests` 中 `status="ended"` 的任务，完成并领取奖励
- 任务链是前期重要的资源和经验来源

### Step 2: 城市建筑
- 检查 `buildingQueue` 长度是否 < 4（有空位）
- 如果有空位，按优先级选择建筑排队建造（可一次排满4个）
- 如果有未解放的建筑需要解放，安排空闲英雄去解放
- 检查 `buildings.liberate` 列表确定哪些建筑可以解放

### Step 3: 招募部队与军队整编
- 通过 `getContent` 获取 `RecruitmentFrame` 查看可招募数量
- 优先招募高等级兵种
- **升级基础版兵种：** 如果有进阶招募建筑（upgraded.available=true），把所有基础版兵团升级为进阶版（REGION_RECRUIT 第4段传 unitStackId）
- **合并同类兵团：** 检查英雄和驻军，用 `unitStackAction FUSION` 合并所有 unitEntityId 相同的兵团。不允许一个英雄身上有两组相同兵种
- 确保驻军有足够预备兵力供英雄战斗后补充

### Step 4: 英雄管理
- 检查酒馆是否有新英雄可以招募（`getContent TavernFrame`）
- 酒馆英雄不理想时考虑花金币刷新（`tavernAction REROLL`），前期慎用
- 空闲英雄有技能点时安排学习职业或技能
- 属性太差或多余的英雄可以解散（`heroAction DISBAND_HERO`）
- 确保所有英雄都有事做（不要闲置）

### Step 5: 矿产升级（英雄空闲时优先）
- **升矿用英雄，和建筑队列并行，不冲突**
- 金矿 ROI 最高：Lv1→2 = +2,000金/天，1.5天回本
- 始终用 IMPROVE_MINE（不要用 UPGRADE_MINE）
- 英雄需要没有部队才能升矿（先 MOVE_TO_REGION 卸载部队）
- 详见 $SKILL_DIR/economy.md 矿产升级章节

### Step 6: 区域控制与扩张侦察
- 用英雄攻击周边 NPC 区域（Zone），扩大领土、占领矿产
- 参考 combat 篇了解兵种克制和战力计算
- 先打弱的 NPC，逐步向外扩张
- **检查开城条件**：`player.currentCityCount < player.maxCityCount` 时可以开新城
  - 城市上限 = max(统治力等级, 财富等级, 荣耀等级)，由排名等级决定
  - 不满足时检查 `nextDominationObjective` / `nextWealthObjective` / `nextHonorObjective`，优先提升最接近下一级的维度
- **本区域 Zone 大部分占领后**：
  1. 用 `getContent WorldMap` 侦察周边空 Region
  2. 用 `scanRegionMines` 查看候选区域的矿产（金矿 + 阵营稀有资源优先）
  3. 选最优位置开城（详见 $SKILL_DIR/territory.md）

### Step 7: 等待动作完成
- 检查 `timeline.attachedMasterActionList` 中所有进行中的动作
- 找到最早完成的动作的 `endDate`
- 等待该动作完成后重新获取 gameState，继续循环

## 装备管理

英雄装备（Artefact）是提升战斗和经济效率的重要手段。每个英雄有 8 个装备槽，装备等级之和不能超过英雄等级（上限 40）。

**装备来源：** 城市仓库（`regionArtefacts`）、英雄背包（`backpackArtefacts`）。通过 `getContent HeroFrame <heroId>` 查看。

**游戏循环中的装备检查（Step 4 之后）：**
1. 检查英雄是否有空装备槽（`equipedArtefacts` 中 position 1-8 未满 8 件）
2. 检查城市仓库（`regionArtefacts`）或背包（`backpackArtefacts`）是否有可用装备
3. 优先装 lvl3 装备（英雄攻击%、NPC卖价、黄金收入效果）
4. 确认装备冷却状态（`unbindDate` 是否过期）

详见 `$SKILL_DIR/equipment.md` — 完整装备效果列表、API 操作方法、冷却机制说明。
